# Technical Flow

This page explains Olympus from system architecture to contract execution and risk math.

## System Architecture

Olympus is designed as a layered system aligned with the current production-style stack:

- Frontend Layer (`olympus-fe`)
  - User interface for My Balances, Earn, DEX Pools, Credit Line, Bridge, and Activity Feed.
  - Reads on-chain state via `viem` and `wagmi`.
  - Reads backend state (prices, guardian logs, protocol snapshots) from API endpoints.
- Backend Intelligence Layer (`olympus-be`)
  - Runs seven role workers: Reporter, Defender, Strategist, Auditor, Accountant, VaR Calculator, and Bridge Manager.
  - Ingests market prices from Pyth/Hermes and computes risk-state outputs.
  - Publishes protocol activity and writes selected outputs on-chain.
- Smart Contract Layer (`olympus-sc`)
  - Core protocol: OlympusSwap, OlympusLend, OlympusRouter, LVRHook, and per-asset OlympusVault contracts.
  - Oracle and cache contracts: OlympusWethOracle, OlympusWbtcOracle, OlympusDotOracle, OlympusDataCache.
  - Cross-chain bridge contracts: OlympusBaseBridge (Base Sepolia) and OlympusPolkadotReceiver (Polkadot Hub TestNet).
- Agent Layer (`olympus-agent`)
  - AI copilot runtime that reads Olympus backend context and supports Polkadot-native operational actions.

## Core Technical Concepts

- Delta-Neutral Vault Structure
  - Users deposit base assets (`WETH`, `WBTC`, `DOT`) into per-asset vaults.
  - Vault strategy targets yield while controlling directional risk via leverage management.
- Spot + Oracle Risk Awareness
  - Oracle prices are synchronized from Pyth/Hermes into Olympus oracle contracts.
  - OlympusSwap spot is compared against oracle references to detect divergence.
- Autonomous Risk Management
  - Defender adjusts dynamic fees during abnormal volatility/divergence windows.
  - Strategist and Auditor maintain leverage and safety bounds through rebalance and guard logic.
- Off-Chain Computation, On-Chain Settlement
  - Heavy analytics (risk scoring, NAV, VaR, policy decisions) run off-chain.
  - Final state transitions and key feeds are committed on-chain.
- Modular Observability
  - Reporter and other roles emit structured logs for user-facing transparency.

## Contract Architecture

Main contracts are organized by responsibility:

- Factory and Router
  - Factory and router contracts coordinate market modules and protocol routing.
- Vaults (`afWETH`, `afWBTC`, `afDOT`)
  - Handle user deposit and withdrawal flows.
  - Maintain share accounting and total assets.
- Execution and Fee Defense
  - OlympusSwap is the execution venue.
  - LVRHook manages adaptive fee updates.
- Credit and Leverage Layer
  - OlympusLend provides delegated USDC credit path used by vault strategy flows.
- Oracle and Risk Cache
  - Per-asset spot oracle contracts store synchronized market references.
  - OlympusDataCache stores on-chain NAV and VaR feed updates.
- Hyperbridge / XCM Onboarding
  - OlympusBaseBridge receives bridge requests on Base Sepolia.
  - OlympusPolkadotReceiver validates routes and triggers destination vault deposit flow on Polkadot Hub TestNet.

For deployed addresses, see **[Smart Contracts](/docs/dev/smart-contracts)**.

## Mathematical Foundation

Core formulas used in protocol logic:

- Share Price
  - `sharePrice = totalAssets / totalSupply`
  - Used for vault accounting and position valuation.
- Position Value (USD)
  - `valueUsd = assetAmount * referencePriceUsd`
  - Reference price comes from synchronized oracle state.
- Divergence Signal
  - `divergencePct = ((spotPrice - referencePrice) / referencePrice) * 100`
  - Used by Defender as a fee-defense input.
- Health Factor (conceptual)
  - `HF = (collateralValue * liquidationThreshold) / debtValue`
  - Used for risk band checks and guard decisions.
- NAV Update Principle
  - `NAV = assets - liabilities`
  - Computed off-chain, then published on-chain for efficient reads.
- VaR (one-sided confidence model)
  - `VaR = quantile(lossDistribution, confidenceLevel)`
  - Computed from recent market history and written to on-chain cache.
